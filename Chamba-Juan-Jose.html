GitHub y Manejo de Datos de Usuario

1. GitHub como herramienta de desarrollo

GitHub no solo sirve para guardar código, también es una herramienta clave en el desarrollo profesional.

  • Commits: registros de cambios realizados en el proyecto
  • Branches: permiten trabajar sin afectar el proyecto principal
  • Pull requests: revisión y unión de cambios
  • Issues: organización de errores o mejoras

GitHub permite trabajar de forma ordenada, documentar avances y colaborar con otros desarrolladores.

2. Manejo de datos del usuario

Frontend

  • Se ejecuta en el navegador
  • Usa cookies o almacenamiento local
  • Más rápido pero menos seguro

Backend

  • Se ejecuta en el servidor
  • Maneja sesiones y bases de datos
  • Más seguro y controlado

3. Cookies

  • Expires / Max-Age: duración
  • Secure: solo HTTPS
  • HttpOnly: bloquea acceso desde JS
  • SameSite: protege contra CSRF

Ejemplo: Usuario con sesión iniciada e idioma español.

4. Sesiones

  • ID único
  • Almacenamiento en servidor
  • Tiempo de expiración

Permiten mantener la sesión activa sin pedir login en cada acción.

5. Cierre de sesión

  • Invalidar sesión
  • Borrar datos
  • Evitar reutilización

Importante en bancos, redes sociales y tiendas online.

6. Problemas comunes

  • Guardar contraseñas en cookies
  • No cerrar sesiones
  • No proteger cookies
  • No usar cifrado

7. Buenas prácticas

  • Usar sesiones para datos sensibles
  • Limitar duración de cookies
  • Cierre automático
  • Validar identidad del usuario

Conclusión

GitHub es una herramienta esencial para el desarrollo web moderno. El manejo adecuado de cookies y sesiones permite proteger la información del usuario. Aplicar buenas prácticas garantiza aplicaciones seguras y confiables.
